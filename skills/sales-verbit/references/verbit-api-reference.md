# Verbit API Reference

## Overview

Verbit's APIs provide programmatic access to "end-to-end transcription and captioning service." Five main API categories:

1. **Insights API (Gen V)** — AI-powered content generation from captions/transcripts
2. **Live Booking API** — Real-time captioning, translations, and transcription
3. **Search API** — Build searchable indexes for caption/transcription assets
4. **Caption Control API** — Session management and caption configuration
5. **Post-Production API** — Transcription and captioning for pre-recorded media

## Authentication

### OAuth 2.0 Bearer Token Flow

**Step 1: Generate API Token**
- Admin role required
- Access via account settings page

**Step 2: Obtain Bearer Token**

```
POST https://users.verbit.co/api/v1/auth
Content-Type: application/json

{
  "data": {
    "api_key": "<API-KEY>"
  }
}
```

**Response:**
```json
{
  "token": "eyJraWQiOiJIMVVkZUVXeGJxYjlHY2R5VjB3WFJyVWpzRG5nNUk3aGpwQUFGWHo3eGwwPSI...."
}
```

**Token Validity:** 24 hours after creation

**Step 3: Use Bearer Token in Requests**

```
GET https://api.verbit.co/api/profiles
Accept: application/json
Authorization: Bearer <TOKEN>
```

### API Key Auth (Post-Production only)

Post-Production APIs support API key authentication via query parameters as an alternative to bearer tokens.

## Insights API (Gen V)

### Get Insights
- **Method:** GET
- **Path:** `/api/v2/insights`
- **Description:** Get insights generated by the Verbit AI engine. Supports insights for both live sessions and post-production jobs.

**Query Parameters:**
- `job_id` (integer, optional): one of [job_id, job_uuid, order_id] must be provided
- `job_uuid` (uuid, optional): one of [job_id, job_uuid, order_id] must be provided
- `order_id` (uuid, optional): one of [job_id, job_uuid, order_id] must be provided
- `insight_name` (string, enum, optional): if not provided all insight types returned
  - Values: chapters, document, person, place, quiz, summary, title, keywords, inconsistencies, index_transcript

**Response Codes:** 200, 400, 401, 403, 422, 500

### Generate Now
- **Method:** POST
- **Path:** `/api/v2/insights/generate`
- **Description:** Generate insights for a job based on the job's insights configuration.

**Query Parameters:**
- `job_id` (integer, optional)
- `job_uuid` (uuid, optional)
- `order_id` (uuid, optional)

**Request Body:** GenerateNowRequestModel with insights object

**Response Codes:** 200, 400, 401, 403, 422, 500

## Live Booking API

### Orders

**List Orders**
- `GET /api/v2/orders`
- Auth: Bearer token

**Create Order**
- `POST /api/v2/orders`
- Auth: Bearer token
- Body: Order configuration with input, output, contact details, settings

**Update Order**
- `PATCH /api/v2/orders`
- Auth: Bearer token
- Body: Array of order objects with updates

**Get Single Order**
- `GET /api/v2/orders/{order_id}`
- Auth: Bearer token

**Get Order Transcript**
- `GET /api/v2/orders/{order_id}/transcript`
- Auth: Bearer token
- Response: Link to transcript, Verbitext availability status

**Cancel Order**
- `PUT /api/v2/orders/{order_id}/status`
- Auth: Bearer token
- Body: `{"name":"canceled"}`

### Recurrence

**Update Recurrence Orders**
- `PATCH /api/v2/recurrences/{recurrence_id}`
- Auth: Bearer token
- Filters: `recurrence`, `order_id` parameters

**Cancel Orders by Recurrence**
- `PUT /api/v2/recurrences/{recurrence_id}/cancel`
- Auth: Bearer token

### Downloads

**Create Download**
- `POST /api/v2/deliveries`
- Parameters: `order_ids`, `formats`, `languages`, delivery options

**Get Download**
- `GET /api/v2/deliveries/{delivery_id}`

**Get Download Status**
- `GET /api/v2/deliveries/{delivery_id}/status`

### Attachments

**Get Pre-signed URL**
- `GET /api/v2/uploads/attachments`
- Query parameter: `file_name`
- Response: UUID, URL, form fields for S3 upload

## Post-Production API

### Jobs Management

**Get job list**
- `GET /jobs`

**Create new job**
- `POST /job/new`
- Sets job name and profile with optional parameters

**Add asset**
- `POST /job/add_asset`
- Uploads media file for use in jobs

**Add media**
- `POST /job/add_media`
- Attaches media via asset ID or direct URL

**Add backup record**
- `POST /job/add_backup_record`
- Includes backup recording for job

**Add attachment notes**
- `POST /job/add_notes_attachments`
- Adds notes and supporting documents

**Add transcription**
- `POST /job/add_transcription`
- Initiates transcription processing

**Start processing**
- `POST /job/perform_transcription`
- Begins ASR and human review workflow

**Get job information**
- `GET /job/info`
- Retrieves current job status and details

**Download draft transcription**
- `GET /job/draft`
- Returns draft content before completion

**Download transcription**
- `GET /job/get_caption`
- Returns completed caption file in selected format

**Get job keywords**
- `GET /job/get_keywords`
- Extracts keyword data from job

**Request review**
- `POST /job/request_review`
- Submits job for professional human review

### SmartPlayer

**Create Smart Player for job**
- `POST /job/smart_player`

**Get job Smart Player**
- `GET /job/smart_player`

**Update Smart Player for job**
- `PUT /job/smart_player`

### Profiles & Users

**List profiles**
- `GET /profiles`

**List users**
- `GET /users`

**Get user by ID**
- `GET /users/{id}`

**Find user by email**
- `GET /users/find`

## Notes

- **Rate limits:** Not published. Implement conservative throttling with exponential backoff.
- **SDKs:** No official client libraries. REST-only.
- **Webhooks/callbacks:** Not documented in public API docs. Poll job status for completion.
- **Base URL:** `https://api.verbit.co/api/`
- **Auth endpoint:** `https://users.verbit.co/api/v1/auth`
